# Experimento E1 — Baseline FL + CIC-IDS2017

**Data:** Junho de 2026
**Status:** Concluído
**Duração:** 167,89 segundos (≈ 2,8 minutos)

---

## 1. Objetivo

Estabelecer um baseline de detecção de intrusão usando Federated Learning com o dataset CIC-IDS2017, sem integração SDN. O modelo deve classificar fluxos de rede como **Benign (0)** ou **Attack (1)** de forma distribuída, sem que os dados de tráfego saiam dos clientes.

---

## 2. Configuração do Experimento

| Parâmetro | Valor |
|---|---|
| Framework FL | Flower (flwr) |
| Modelo | MLP (IDSNet) |
| Dataset | CIC-IDS2017 |
| Algoritmo de agregação | FedAvg |
| Número de clientes | 2 |
| Rodadas | 5 |
| Épocas por rodada | 2 |
| Batch size | 256 |
| Otimizador | Adam (lr = 0.001) |
| Loss | CrossEntropyLoss |
| Particionamento | Non-IID |

## 4. Modelo

### Arquitetura (IDSNet)

```
Input (51 features)
    → Linear(51 → 128) → ReLU → Dropout(0.3)
    → Linear(128 → 64) → ReLU → Dropout(0.2)
    → Linear(64 → 2)
    → Softmax → [Benign | Attack]
```

### Features utilizadas (51 no total)

Features de volume, temporização, flags TCP, tamanho de pacotes e janelas de transmissão extraídas dos fluxos de rede, incluindo: `Flow Duration`, `Flow Bytes/s`, `Flow Packets/s`, `SYN Flag Count`, `Init Fwd/Bwd Win Bytes`, `Active Mean`, `Idle Mean`, entre outras.

## 6. Análise

### Pontos positivos

- **Precisão de 99,16%** — menos de 1% de falsos positivos. O modelo raramente classifica tráfego legítimo como ataque, o que é essencial para operação real de um IDS.
- **Convergência rápida** — o loss caiu 30% do round 1 ao round 4 e estabilizou no round 5, indicando que 5 rodadas são suficientes para este setup.
- **Consistência entre clientes** — apesar do particionamento Non-IID, os dois clientes convergiram para métricas similares, validando a robustez do FedAvg com dados heterogêneos.
- **Zero falhas de comunicação** — todas as 5 rodadas completaram com `0 failures`.
- **Privacidade preservada** — os 2.313.810 registros de tráfego nunca saíram dos clientes. Apenas os pesos do modelo (51×128 + 128×64 + 64×2 parâmetros) trafegaram pela rede.

### Ponto de atenção

- **Recall de 92,73%** — aproximadamente 7,3% dos ataques reais não são detectados (falsos negativos). Em um IDS operacional, recall alto é mais crítico que precisão. Melhoria planejada: aplicar peso de classe (`CrossEntropyLoss(weight=[1.0, 3.0])`) para penalizar mais os falsos negativos.

### Comparação com treino centralizado (referência)

O resultado de F1 ≈ 95,8% é competitivo com abordagens centralizadas reportadas na literatura para o CIC-IDS2017, que tipicamente variam entre 94% e 99% dependendo do modelo e das features utilizadas — mas sem as garantias de privacidade que o FL oferece.

---

## 7. Falhas e limitações

- O dataset CIC-IDS2017 tem desbalanceamento severo (85,5% Benign), o que favorece artificialmente a acurácia. F1-score e recall são as métricas mais confiáveis para avaliar o desempenho real.
- A classe `Infiltration` tem apenas 36 amostras — praticamente invisível para o modelo neste setup.
- O experimento foi executado em CPU, com os dois clientes na mesma máquina — não representa latência de rede real entre nós distribuídos.
